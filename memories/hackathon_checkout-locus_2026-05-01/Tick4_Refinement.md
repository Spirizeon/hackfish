# Tick 4: Refinement

## Team Iterations

### AccessPay (Team 1)
**Pivot**: "We need to test with actual visually impaired users"
- **Refined focus**: Partner with **Vision Australia** for user testing
- **Tech pivot**: Use Locus accessibility-first checkout SDK
- **Differentiation**: WCAG AAA compliance - first payment app to achieve it
- **Demo**: Live screen reader demo showing checkout flow
- **Sponsor integration**: Locus CheckoutWithLocus + Accessibility API

### InvoiceFlow (Team 4)
**Pivot**: "Compliance first - we need to partner with licensed factors"
- **Refined focus**: Start with **invoice verification** (not financing)
- **Tech pivot**: Locus payment verification API + AI fraud detection
- **Differentiation**: Real-time credit risk scoring using Locus transaction data
- **Demo**: Live invoice upload → instant risk score
- **Sponsor integration**: Locus Payment API, Banking API

### CheckoutKit (Team 3)
**Pivot**: "Make it truly drop-in - less is more"
- **Refined focus**: 3 core components: PayButton, CheckoutForm, OrderSummary
- **Tech pivot**: Locus pre-built UI components wrapped in React
- **Differentiation**: 40% better conversion than generic checkout
- **Demo**: Side-by-side conversion test
- **Sponsor integration**: Locus Checkout SDK

### PayFlow API (Team 2)
**Pivot**: "Narrow to one vertical - we chose SaaS subscriptions"
- **Refined focus**: Subscription billing for SaaS companies
- **Tech pivot**: Locus subscription API + multi-currency
- **Differentiation**: Handle failed payments automatically (dunning)
- **Demo**: Live subscription setup with automatic retry
- **Sponsor integration**: Locus Subscription API, Retry API
